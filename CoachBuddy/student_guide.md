# 프로젝트 개요: CoachBuddy

## 1. 서비스 기본 정보
- 서비스명: CoachBuddy (1인 강사를 위한 AI 코칭 피드백 파트너)
- 핵심 가치:  
  강사가 학생 영상을 업로드하면, AI가 분석 데이터를 추출하고  
  전문적인 코칭 피드백 시트를 자동으로 생성하는 웹 서비스
- 현재 상태:  
  - 코치용 핵심 로직이 이미 구현 완료
  - 다음 단계는 “학생용 포털(student.html) 구축 + 실제 수익화 기능 연동”

## 2. 디자인 컨셉
- 테마: [Variant 1: Professional & Clean]  
  - 메인 컬러: 블루 / 틸(Teal) 계열
  - 인터페이스: 깔끔, 직관적, 코치/학생 모두가 편하게 사용할 수 있도록
  - 기준:  
    - 코치 대시보드와 일관된 UI/UX로, 색상·폰트·카드 스타일을 동일 템플릿으로 재사용
    - 모바일에서도 카드형 레이아웃으로 볼 수 있도록 반응형 유지

---

## 3. 현재 완료된 코치용 기능

### 3.1. 대시보드 및 내비게이션
- 메뉴 구성:
  - 대시보드
  - 학생 관리
  - 리포트
  - 프리미엄
- 내비게이션:
  - 상단 메뉴 클릭 → 각 섹션으로 부드러운 스큌롤 이동 (Smooth Scrolling)
- 학생 관리:
  - 코치가 관리 중인 학생들의 카드형 목록 제공
- 리포트:
  - 전체적인 분석 통계 및 성장 추이를 한눈에 확인 가능한 대시보드

### 3.2. 스마트 업로드 시뮬레이션
- UI:
  - 드래그 앤 드롭 또는 클릭으로 영상 업로드
- 로직:
  - 업로드 시 js/app.js에서 로딩 스피너 + “AI 분석 중…” 표시
  - 약 2.5초 후 analysis.html로 자동 이동

### 3.3. AI 동작 분석 결과 페이지 (analysis.html)
- 이미지 분석:
  - 농구 자유투를 예시로 한 AI 스켈레톤 추출 이미지 표시
- 정밀 메트릭:
  - 슛 각도(엘보우), 릴리즈 아크, 점프 높이, 신체 대칭성 등 물리적 지표 수치화
- AI 코멘터리:
  - 분석 데이터를 기반으로 “개선 방향 + 칭찬 포인트”를 자연스럽게 표현한 텍스트 제공

## 4. 다음 단계: Student Portal + Monetization 구현

### 4.1. 학생 전용 포털 (student.html)

목표:  
학생이 자신의 분석 리포트를 확인하고, 성장 과정을 추적하는 **개인화된 대시보드**를 제공.

요구사항:

1) 페이지 구조:
   - URL: `/student.html`
   - 레이아웃:  
     - 상단 네비게이션(로그인/마이페이지, 코치/학생 구분 표시)  
     - 메인:  
       - 최근 분석 결과 카드  
       - 그래프(성장 추이: 릴리즈 아크, 점프 높이, 대칭성 등)  
       - 코치 코멘트 섹션  
       - 추천 트레이닝/운동 루틴 카드

2) 기능:
   - 최근 분석 리포트 목록(최근 5개까지)
   - 클릭 시 해당 분석 페이지(analysis.html)로 이동
   - 그래프:
     - 라인 차트: 시간에 따른 주요 지표 변화
     - 색상: 향상은 파랑/틴색, 저조는 빨강/노랑으로 시각화
   - 코치 코멘트 표시:
     - 코치가 작성한 텍스트형 코멘트 표시
   - 추천 루틴:
     - AI가 “개선 포인트”에 따라, 간단한 운동 리스트(예: 스쿼트, 런지, 코어 훈련 등)를 제안

3) UI/UX:
   - 코치 대시보드와 동일한 틸(Teal) 테마 + 폰트 스타일
   - 카드형 레이아웃, 반응형
   - 모바일에서 손가락으로 스와이프하여 최근 리포트 확인 가능

---

### 4.2. 수익화(Monetization) 기능

학생/코치가 모두 쓰는 서비스를 기반으로, **3가지 수익화 채널을 연동**하고자 합니다.

#### 1) AI 추천 샵 (Affiliate 수익)

- 목적:  
  분석 결과에서 “부족한 부분”을 감지하고,  
  그에 맞는 보충제나 장비를 추천하여 쿠팡/네이버 스토어 등으로 연결 → 수수료 수익
- 위치:
  - student.html:  
    - 분석 리포트 하단에 “추천 상품” 카드 섹션
  - analysis.html:  
    - 메트릭 분석 하단에 “AI 추천 상품” 카드
- 구현 방식:
  - 분석 결과(예: `하체 근력 부족`, `체중 관리 필요`, `구기운동 보호구`)에 따라  
    키워드를 기반으로 상품 카드를 구성
  - 예시:
    - `하체 근력 부족` → 프로틴, 보충제 키워드
    - `구기운동 보호구` → 무릎 보호대, 테이핑, 헤드밴드 등
  - 카드:
    - 상품 이름
    - 간단한 설명(예: “하체 근력 강화에 도움을 주는 단백질 보충제”)
    - 이미지(더미 이미지라도, 실제 상품 썸네일 자리로)
    - “쿠팡/스토어 링크” 버튼(실제 URL은 더미 링크)
- 마크업:
  - `<a>` 태그로 affiliate 링크 위치를 표시만 해두면 OK  
    (예: `// Affiliate link: PROD_001` 등의 주석)

#### 2) 프리미엄 콘텐츠 (유료)

- 목적:  
  고퀄리티 교정 운동 영상, 루틴을 유료로 판매하거나 구독으로 제공
- 위치:
  - student.html:  
    - “프리미엄 콘텐츠” 섹션
  - analysis.html:  
    - “코치 추천 프리미엄 콘텐츠” 카드
- 구현:
  - 카드형 리스트:
    - 콘텐츠 제목(예: “자유투 개선 퍼팩트 트레이닝”, “하체 폭발 훈련”)
    - 난이도: 초급, 중급, 고급
    - 재생 시간/예상 효과 간단 소개
    - “구매/구독하기” 버튼
  - 실제 결제 연동은 나중에 Toss, 카카오페이 등으로 대체 예정이므로,  
    버튼은 “구매하기 (예정)” 또는 “구독하기 (예정)”처럼 더미 표기

#### 3) Pro 플랜 (무제한 분석 + PDF 리포트)

- 목적:  
  - 코치/학생 모두에게  
    무제한 분석 + PDF 리포트 다운로드를 유료로 제공
- 위치:
  - student.html:  
    - 상단 또는 측면에 “Pro 플랜” 안내 섹션
  - analysis.html:  
    - “PDF 리포트 다운로드” 버튼
- 기능:
  - 기본 플랜:
    - 분석 3회, PDF 리포트 1회 다운로드 제한
  - Pro 플랜:
    - 무제한 분석 + PDF 리포트 다운로드
  - 버튼:
    - “Pro 플랜으로 업그레이드” 버튼
    - “PDF 리포트 다운로드” 버튼(Pro 플랜일 경우에만 활성화)

## 5. 요청하는 구현 범위

아래 내용을 기반으로 **student.html + analysis.html + js/app.js**에 대한 코드를 생성해 주세요.

1) student.html 구조 및 UI
   - 학생용 대시보드 페이지
   - 최근 분석 결과 카드, 라인 그래프(최소 더미 데이터 사용)
   - 코치 코멘트, 추천 루틴 카드
   - 코치용 대시보드와 같은 틸 테마/디자인 유지

2) 수익화 UI 구성
   - AI 추천 상품 카드
   - 프리미엄 콘텐츠 카드
   - Pro 플랜 안내/업그레이드 버튼
   - 버튼/링크 위치만 구현 (실제 결제·API 연동은 나중에)

3) 필요 시 js/app.js에
   - 학생용 데이터(예: latestAnalysis, growthMetrics)를 더미 객체로 정의
   - 카드/그래프를 렌더링하는 기본 로직 추가

4) 디자인 일관성
   - 코치 대시보드의 카드/버튼/폰트/색상 톤을 그대로 student.html에 반영
   - 반응형 유지(모바일에서도 카드형 레이아웃)

## 6. 안티그래비티에게 주는 작업 명령 예시

이 프롬프트를 안티그래비티에 준 뒤, 아래처럼 순차적으로 요청해도 됩니다.

- 1) “student.html 구조와 UI를 코치 대시보드와 동일한 틸 테마로 만들어줘”  
- 2) “student.html에 학생용 최근 분석 리포트 카드, 그래프, 코치 코멘트, 추천 루틴 카드를 넣어줘”  
- 3) “수익화 요소: AI 추천 상품, 프리미엄 콘텐츠, Pro 플랜을 student.html과 analysis.html에 UI로만 구현해줘”  

이렇게 하면,  
- 코치용 코어 기능은 그대로 유지하면서  
- 학생용 포털 + 실제 수익화 요소를 **연동 가능한 UI/구조**로 확장할 수 있습니다.
