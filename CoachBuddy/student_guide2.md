# CoachBuddy - 다음 단계: 실제 수익화 준비 및 운영 구조

## 1. 현재 상태 요약

- 학생 전용 대시보드(student.html) 완성:
  - 최근 분석 결과 카드
  - 시간에 따른 성장 그래프(더미 시각화)
  - 코치 코멘트
  - AI가 분석 포인트에 맞춘 추천 루틴 카드

- 수익화 UI 구현:
  - AI 추천 샵: 분석 결과와 연동된 보충제/장비 제휴 카드(쿠팡/스토어 링크 버튼)
  - 프리미엄 레슨: '구매/잠금' UI가 적용된 유료 강의 리스트
  - Pro 플랜: PDF 리포트 다운로드 버튼에 Pro 배지

- 모드 전환:
  - 모든 페이지 헤더 우측에 [코치 모드 / 학생 모드] 전환 버튼 추가

- 동적 데이터 렌더링:
  - js/app.js에 더미 데이터(latestAnalysis, growthMetrics, shopItems, premiumLessons) 추가
  - 페이지 로드 시 학생 포털, 추천 상품, 프리미엄 강의 등이 자동 렌더링

이제는 **이 UI 구조를 기반으로 실제 수익화 및 운영 가능한 구조로 확장**하는 작업을 진행하고자 합니다.

## 3. 1단계: Pro 플랜 + PDF 리포트(Pro용) 강화

### 3.1. PDF 리포트 페이지/구조 설계
- `/report.html` 또는 `analysis.html` 내부에 **PDF 리포트 출력용 섹션**을 명확히 정의
- PDF 리포트 구성 예시:
  - 학생 정보(이름, 나이, 포지션 등)
  - 분석 날짜
  - 주요 메트릭 표(점프 높이, 릴리즈 아크, 대칭성, 슛 각도 등)
  - 그래프(이미지/캔버스로 렌더링된 라인 차트)
  - 코치 코멘터리
  - 추천 루틴
- js/app.js에서:
  - `generatePDF()` 또는 `downloadReportPDF()` 함수를 만들어,
    - HTML 요소를 `html2pdf.js` 또는 `jsPDF` 스타일로 PDF로 변환하는 더미 흉내를 낸다.
  - Pro 플랜이 아닐 경우,
    - “PDF 다운로드는 Pro 플랜에서만 가능합니다” 문구 표시
  - Pro 플랜일 경우,
    - “PDF 생성 중…” → “보고서 다운로드 완료”처럼 단계형 UI

### 3.2. Pro 플랜 상태 구현
- js/app.js에 `isPro` 또는 `subscriptionPlan` 변수를 두고,
  - `isPro = false` 기본값
  - 버튼/섹션에 따라 Pro/비Pro 시에 표시·비표시를 제어
- UI 예시:
  - Pro 배지: badge.pro-badge (텍스트: “Pro”, 스타일: 틸/블루)
  - Pro 전용 버튼: `btn-pro-only`
  - Pro 전용 카드: `card-pro-only`
- Pro 플랜일 때는:
  - PDF 다운로드 버튼 활성화
  - 수제한 프리미엄 콘텐츠 잠금 해제
- Pro 아닐 때는:
  - PDF 버튼에 문구/툴팁
  - 일부 프리미엄 루틴/영상에 “잠금” 레이어

## 4. 2단계: 프리미엄/잠금 레이어 구조 강화

### 4.1. 코치/학생용 프리미엄 레이어 UI
- Premium Lessons(프리미엄 레슨):
  - Card 내부에 `.lock-overlay` 레이어를 추가
  - 비 Pro 플랜일 경우:
    - 어두운 overlay + “Pro 플랜에서 이용 가능” 텍스트
  - Pro 플랜일 경우:
    - 레이어 사라짐
- Student Portal:
  - “추천 루틴”과 “프리미엄 루틴”을 구분
  - “Pro 전용 루틴” 카드에는 Pro 배지

### 4.2. 잠금 해제 유도 문구
- Pro 배너 섹션에:
  - “무제한 분석 + PDF 리포트 다운로드”
  - “모든 프리미엄 레슨을 Pro 플랜으로 열어보세요”
  - “월 1만 원으로 시작하는 Pro 플랜” 등의 텍스트
- 버튼:
  - “Pro 플랜으로 시작하기(예정)”
  - “결제 페이지로 이동(예정)”

## 5. 3단계: 결제·수익화 흐름 구조 설계

실제 결제 연동은 나중에 하되, **구조만 먼저 준비**하는 방향으로 정의해 주세요.

1) 가상 프로세스 설계
   - 학생/코치가:
     - Pro 플랜 또는 특정 프리미엄 레슨을 선택
   - “결제 예정” 팝업/모달로 이동
   - 모달 내에:
     - 결제 금액(예: Pro 플랜 월 10,000원)
     - 카드/카카오페이/Toss 표시(아이콘만)
     - “결제하기(예정)” 버튼(더미 이벤트)

2) js/app.js에 더미 결제 흐름
   - `startPayment(productId)` 함수:
     - productId에 따라 Pro 플랜 / 프리미엄 레슨을 구분
     - `alert("결제 페이지로 이동합니다(예정)")` 또는
       - `isPro = true`로 설정하는 더미 흉내
   - 콘텐츠 잠금 해제:
     - Pro 활성화 시 모든 Pro 전용 요소를 show로 변경

3) 결제 연동 준비를 위한 파일 구조
   - `src/` 아래에 가상 API 구조를 만들어 두면 좋습니다:
     - `/api/subscription/checkout.js` (더미)
     - `/api/subscription/verify.js` (더미)
   - 이는 향후 Toss/KakaoPay 연동 시 그대로 활용 가능

## 6. 4단계: 코치 대시보드에 리포트/결제 관리 영역 추가

코치 입장에서 학생의 프로필·결제·리포트 현황을 볼 수 있는 섹션을 추가합니다.

1) 학생 프로필 관리
   - `학생 관리 > [학생]` 클릭 시:
     - 프로필 페이지(예: `/student-profile.html?id=1`)
     - 그 안에:
       - 최근 분석 리포트 카드
       - 결제 상태(예: “Free / Pro”, “결제 예정”)
       - 코치 코멘트

2) 리포트·결제 관리 섹션
   - `리포트` 탭 내부에:
     - 전체 학생의 Pro 플랜 현황(예: 그래프나 표)
     - “Pro 플랜 사용자 비율” 카드
     - “가장 많이 본 프리미엄 레슨” 카드
   - 데이터는:
     - 더미 배열 `proStudents`, `popularLessons`로 구현
