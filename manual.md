다음은 1인 기업을 위한 수익화 앱을, NotebookLM MCP + Stitch MCP를 활용해 안티그래비티가 자동으로 선정·디자인·제작하는 흐름을 정의한 프롬프트입니다.

# 1. 프로젝트 목표
- 1인 사업자(프리랜서, 소상공인, 콘텐츠 크리에이터 등)가 직접 쓰고, 
  월 5,000~30,000원 수준으로 구독할 수 있는 웹 기반 수익화 앱을 만드는 것
- 최종 산출물:
  - 1개 선정 앱 아이디어
  - 3개 디자인 후보 (Stitch MCP로 생성)
  - 1개 승인된 디자인
  - 그 디자인을 기반으로 한 웹 앱 구조 (HTML, CSS, JS, 폴더 구조)

# 2. 워크플로우 단계
안티그래비티는 다음 순서대로 진행합니다.

1) 수익화 앱 아이디어 도출 및 선정 (NotebookLM MCP 사용)
2) 3개 디자인 생성 (Stitch MCP 사용)
3) 사용자 승인: 디자인 3개 중 1개를 선택하면 그 디자인으로 확정
4) 확정 디자인을 기반으로 웹 앱 구조 생성 (HTML/CSS/JS + 폴더 구조)

# 3. 1단계: NotebookLM MCP로 수익화 앱 추천

- NotebookLM MCP를 사용해, 1인 기업 수익화에 적합한 앱 아이디어를 5~10개 추천
- 추천 기준:
  - 타겟: 1인 사업자, 프리랜서, 소상공인, 학생, 크리에이터
  - 매출 모델: 무료 + 유료 프리미엄 기능, 월 구독, 1회 구매, 기능별 잠금 등
  - 기술적 구현 난이도: 웹 기반, GitHub Pages + 클라우드/클라우드 없이도 운영 가능
- 각 아이디어는 다음 형식으로 정리:
  - 앱 이름
  - 대상 유저
  - 핵심 기능 3~5개
  - 수익화 모델
  - GitHub Pages + 클라우드/클라우드 없는 환경에서의 구현 가능도
- 이후 안티그래비티가 5개를 추려서, 사용자에게 1개를 선택하도록 제안
- 사용자가 1개를 선택하면, 그 앱을 최종 앱으로 확정

# 4. 2단계: Stitch MCP로 3개 디자인 생성

- 최종 선정 앱에 대해, Stitch MCP를 사용해 3개 디자인 시안을 생성
- 각 디자인은:
  - 1~2개 주요 화면(예: 홈, 견적서 생성, 프리미엄 안내 등)
  - 모바일/웹 반응형을 고려한 레이아웃
- 3개 디자인을 다음과 같이 정리해 사용자에게 전달:
  - 디자인 1: 용도, 타겟, 시각적 스타일(예: 모던, 미니멀, 과감한 색상)  
  - 디자인 2, 3: 동일 구조
- 사용자는 3개 중에서 1개를 선택해, “디자인 1 / 디자인 2 / 디자인 3” 중 하나를 명시

# 5. 3단계: 사용자 승인 디자인 확정

- 사용자가 “이 디자인으로 한다”고 선택하면,  
  안티그래비티는 그 디자인을 “확정 스타일 가이드”로 사용
- 확정 디자인 기준:
  - 색상 팔레트
  - 폰트 스타일
  - 버튼·카드·레이아웃 구조
  - 반응형(모바일 우선) 방식

# 6. 4단계: 확정 디자인을 기반으로 웹 앱 구조 생성

- 확정 디자인을 기반으로, 웹 앱의 초기 구조를 만듭니다.
- 폴더 구조 예시:
  project-1person-app/
  ├── index.html
  ├── css/
  │   └── style.css
  ├── js/
  │   └── app.js
  └── docs/
      └── 01-plan.md

- index.html:
  - 확정 디자인에 맞는 UI 구조(네비게이션, 메인 섹션, Call-to-Action 등)
- css/style.css:
  - 디자인에 맞는 색상, 폰트, 반응형 레이아웃
- js/app.js:
  - 기본 이벤트 처리(예: 토글, 모달, 폼, 버튼)
- docs/01-plan.md:
  - 앱 목표, 핵심 기능, 수익화 모델, 향후 구현 로드맵

# 7. 안전 및 분리 규칙
- 기존 QuickBill-app, naengjanggo-fitness 등 다른 프로젝트에는 절대 영향을 주지 않음
- 새로운 앱은 완전히 새로운 폴더(project-1person-app 또는 사용자가 지정한 이름)에서만 생성
- NotebookLM MCP와 Stitch MCP는 안티그래비티의 MCP 설정에 이미 등록된 것으로 가정

# 8. 명령어 체계 (사용자가 입력하는 흐름 예시)

1) 안티그래비티에게 이 프롬프트를 준 뒤:
   - “1인 기업 수익화 앱 아이디어를 NotebookLM MCP로 10개 추천해 줘”라고 요청
2) 안티그래비티가 5개를 추려서 보여주면:
   - 사용자: “이 중에서 이 앱으로 하겠다: [앱 이름]”라고 지정
3) 안티그래비티가:
   - “Stitch MCP로 3개 디자인을 생성했습니다. 1, 2, 3 중 하나를 선택해 주세요”라고 출력
4) 사용자: “디자인 2로 한다”라고 응답
5) 안티그래비티는:
   - 사용자가 선택한 디자인을 기반으로 project-1person-app/ 폴더 구조와 코드를 생성

이 프롬프트를 기준으로, 안티그래비티는 NotebookLM MCP로 1인 기업 수익화 앱을 선정하고,  
Stitch MCP로 3개 디자인을 만들고,  
사용자가 승인한 디자인으로 최종 웹 앱 구조를 자동으로 만들어낼 수 있습니다.
